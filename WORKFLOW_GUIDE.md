# 智能体编排模式 — 节点配置完全指南

## 核心概念

工作流 = 节点（Node）+ 连线（Edge），数据沿连线从左→右或从上→下流动。

**每个节点做的事**：接收上游数据 → 处理 → 输出给下游。

---

## 内置变量（所有节点通用）

在输入框中使用 `{{变量名}}` 语法引用：

| 变量 | 来源 | 说明 |
|------|------|------|
| `{{query}}` | 系统注入 | 用户输入的问题文本 |
| `{{_result}}` | 上一步节点 | 上一个节点的输出结果 |
| `{{_reply}}` | 系统 | 最终的回复文本（回复节点设置） |
| `{{knowledge}}` | 知识库节点 | 知识库检索到的内容（取决于 outputVar 设置） |
| `{{userId}}` | 系统 | 当前用户 ID |
| `{{conversationId}}` | 系统 | 当前会话 ID |

> 每个节点执行后，它的输出会自动写入 `{{_result}}`，下一个节点的 `{{_result}}` 就是上一个节点的输出。

---

## 所有节点详解

### 🚀 开始节点
> 工作流入口，**每个工作流必须有一个**，且只能放在最前面。

**无需配置** — 它只是标记起点，不做任何处理。

**连线规则**：只能作为 source（输出端），不能作为 target（输入端）。

---

### 🏁 结束节点
> 工作流出口，收到数据后终止执行。可选。

| 配置项 | 含义 | 示例 |
|--------|------|------|
| 输出变量 | 把结果存入哪个变量名 | `answer`（后续可通过 `{{answer}}` 引用） |

**连线规则**：只能作为 target（输入端）。

---

### 💬 回复节点
> **最重要的输出节点** — 把内容返回给用户。没有这个节点，用户看不到任何回复！

| 配置项 | 含义 | 默认值 | 示例 |
|--------|------|--------|------|
| 回复内容 | 返回给用户的文本，支持 `{{变量}}` | 空 | `{{_result}}` 或 `根据知识库，您有 {{customerCount}} 个客户` |
| 格式 | 输出格式 | 纯文本 | 纯文本 / **Markdown**（推荐，支持标题、表格、加粗等） |

> ⚠️ **常见错误**：回复内容为空 → 用户什么都看不到。务必填入 `{{_result}}` 或自定义文本。

**连线规则**：有输入连接点（上一步连过来），也有输出连接点（可连到结束节点）。

---

### 🧠 LLM 节点
> 调用 AI 大模型生成回答。工作流的"大脑"。

| 配置项 | 含义 | 说明 |
|--------|------|------|
| 模型 | 选择哪个 AI 模型 | 下拉列表按厂商分组，显示在线/离线状态。也可选「使用变量」用 `{{modelId}}` 动态传参 |
| 提示词 | 给模型的系统指令 | **核心配置**，告诉 AI 它的角色、任务、格式要求。支持 `{{变量}}` |
| Temperature | 随机性 0~2 | 0 = 保守精确，1 = 平衡，2 = 天马行空 |

**提示词示例**：
```
你是一个业务数据分析助手。根据以下数据回答问题。

知识库检索结果：
{{knowledge}}

用户问题：{{query}}

要求：
1. 根据数据准确回答，不要编造
2. 涉及数字请明确列出
3. 数据不包含的信息请说"暂无数据"
```

**连线规则**：一个输入 + 一个输出。

---

### 📚 知识库检索节点
> 从指定的知识库中取出文档内容，供 LLM 节点引用。

| 配置项 | 含义 | 默认值 | 说明 |
|--------|------|--------|------|
| 知识库 | 选择哪个知识库 | — | 下拉列表显示知识库名称+ID+状态（已发布/草稿） |
| 返回条数 | 最多取多少篇文档 | 3 | 1~20，越大上下文越多 |
| 相似度阈值 | 匹配精度门槛 | 0.7 | 0~1，越高越严格（只返回高度匹配的内容） |
| 输出变量 | 结果存到哪个变量 | `knowledge` | 在 LLM 的提示词中用 `{{knowledge}}` 引用 |

**典型用法**：知识库 → LLM → 回复
```
用户问"客户有多少"
→ 知识库检出客户表（刘翔、刘备、张飞）→ 存入 {{knowledge}}
→ LLM 提示词里写 {{knowledge}} → AI 读到数据后回答"你有 3 个客户"
→ 回复节点输出 {{_result}}
```

> ⚠️ **已修复的 Bug**：之前知识库作为第一个处理节点时不会执行，现已修复。

**连线规则**：一个输入 + 一个输出。

---

### 🔀 条件判断节点
> 根据条件分流，走不同的路径。

| 配置项 | 含义 | 示例 |
|--------|------|------|
| 条件表达式 | 判断条件，用 `{{变量}}` | `{{_result}} == '成功'` 或 `{{count}} > 10` |

**特殊连线**：条件节点有**两个输出连接点**：
- **true（紫色圆点）**：条件成立走这边
- **false（黄色圆点）**：条件不成立走这边

**典型用法**：
```
开始 → 问题分类 → 条件判断({{_result}}=='技术问题')
                      ├── true → 技术知识库 → LLM → 回复
                      └── false → 通用 LLM → 回复
```

---

### 🔍 问题分类节点
> 让 AI 自动判断用户意图，将问题归类。

| 配置项 | 含义 | 示例 |
|--------|------|------|
| 分类 | 每行一个类别 | `产品咨询`<br>`技术支持`<br>`投诉建议` |

输出存入 `{{_result}}`，通常是匹配到的类目名称。配合条件判断使用。

---

### 🔗 结果合并节点
> 把多条路径的结果合并在一起。

| 配置项 | 含义 | 选项 |
|--------|------|------|
| 合并策略 | 怎么合并 | 取第一个结果 / 拼接所有结果 / 取最优结果 |

**典型用法**：多个分支处理完后汇合到一处。

---

### 🔄 循环节点
> 重复执行某段逻辑直到条件满足。

| 配置项 | 含义 | 默认值 |
|--------|------|--------|
| 循环条件 | 继续循环的条件 | — |
| 最大次数 | 安全上限 | 10 |
| 迭代变量 | 存到哪个变量 | `item` |

---

### 🌐 HTTP 请求节点
> 调用外部 API。

| 配置项 | 含义 | 示例 |
|--------|------|------|
| 请求方法 | HTTP 方法 | GET / POST / PUT / DELETE |
| URL | 接口地址 | `https://api.example.com/data?q={{query}}` |
| Body | 请求体（POST/PUT） | `{"query": "{{query}}"}` |

输出存入 `{{_result}}`，值为接口返回的 JSON 字符串。

---

| 配置项 | 含义 | 默认值 |

## 标准工作流模板

### 模板 1：AI 问答（最常用）
```
开始 → LLM → 回复 → 结束
```
适用于：简单的一问一答。

### 模板 2：知识库问答
```
开始 → 知识库检索 → LLM → 回复 → 结束
```
适用于：基于知识库的精准回答。LLM 提示词中引用 `{{knowledge}}`。

### 模板 3：意图路由
```
开始 → 问题分类 → 条件判断
                    ├── true → LLM_A → 回复
                    └── false → LLM_B → 回复
```
适用于：根据用户意图走不同的处理逻辑。

### 模板 4：联网查询
```
开始 → HTTP请求 → LLM → 回复 → 结束
```
适用于：调用外部 API 获取实时数据，AI 整理后输出。

## 常见踩坑

| 问题 | 原因 | 解决 |
|------|------|------|
| 回复为空 | 回复节点 content 没有填 | 填入 `{{_result}}` |
| 知识库不执行 | 之前有 Bug（已修复），或知识库没选 | 下拉选择知识库 |
| 连线拖不动 | 鼠标悬停节点边缘才出现 Handle | 把鼠标移到节点边框处 |
| LLM 乱答 | Prompt 没有约束 | 写清楚角色、格式要求、数据来源 |
| 节点删不掉 | 选中后按 Delete 键 | 或点配置面板底部的「删除节点」按钮 |
| 保存了但没生效 | 需要重启后端 | 重启 `node src/app.js` |
